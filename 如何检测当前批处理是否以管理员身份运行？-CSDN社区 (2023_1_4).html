如何检测当前批处理是否以管理员身份运行？

DeepBlueTears 2011-08-12 04:47:57
现在我遇到了一个问题：批处理中有部分命令需要管理员身份才能执行，我不想用户等到命令执行失败，被告知需要管理员身份时才想起要以管理员身份运行，那样会浪费很多时间。

我希望能够有办法检测批处理是否以管理员身份运行，从而在第一时间就告知用户。
又或者批处理一被执行的时候，就立即向用户请求管理员权限，从而解决这个问题。

求各位高手帮忙想想办法，或者是提供一些这方面的资料，谢谢。

感谢各位的帮助。虽然还是没有找到好的办法来请求管理员权限，但是已经找到了测试当前批处理是否具有管理员权限的办法。

办法1 写入文件法
向System32文件夹写入临时文件，若能写入成功，则说明批处理具有管理员权限，否则说明不具有。
代码如下：

  2. @ECHO OFF

  4. Rem 创建文件路径
  5. set TempFile_Name=%SystemRoot%\System32\BatTestUACin_SysRt%Random%.batemp
  6. echo %TempFile_Name%

  8. Rem 写入文件
  9. ( echo "BAT Test UAC in Temp" >%TempFile_Name% ) 1>nul 2>nul

  11. Rem 判断写入是否成功
  12. if exist %TempFile_Name% (
  13. echo 正在以管理员身份运行当前批处理
  14. ) else (
  15. echo 没有以管理员身份运行当前批处理 
  16. )
  17. pause

  19. Rem 删除临时文件
  20. del %TempFile_Name% 1>nul 2>nul

  22. echo 按空格键退出……
  23. pause >nul



方法2 窗口标题判断法
以管理员身份运行的批处理，设置标题后，标题前方会提示 "管理员： " ,可判断特定窗口标题所对应的进程名是否为cmd.exe 来确认批处理是否以管理员身份运行。代码如下：


  2. @echo off
  3. title BAT
  4. setlocal EnableDELAYEDEXPANSION

  6. Rem 设置标志变量
  7. set UAC=0

  9. Rem 获取tasklist命令输出
  10. For /f "usebackq delims==" %%i In (`tasklist /fi "WINDOWTITLE eq 管理员: BAT"`) Do (
  11.   set "cmdline=%%i"
  12.   set "cmdline=!cmdline:~0,7!"
  13.   
  14.   if "!cmdline!" == "cmd.exe" ( 
  15.   set UAC=1
  16.   )
  17. )

  19. if "%UAC%" == "1" (
  20. echo 正在以管理员身份运行当前批处理
  21. ) else (
  22. echo 没有以管理员身份运行当前批处理
  23. )

  25. :End
  26. pause
  27. exit



此方法需要外部命令 tasklist.exe 支持才能成功。
